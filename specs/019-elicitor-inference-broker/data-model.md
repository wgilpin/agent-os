# Data Model: Elicitor UDS Transport

This document defines the schemas and data structures for the Elicitor's UDS transport integration.

## 1. System Elicitor Manifest Schema

The substrate creates a virtual `AgentOS.Manifest` for the elicitor with the following parameters:

- **purpose**: `"Elicit specification"`
- **owner**: `"system"`
- **supervision**: `"none"`
- **grants**: `[]`
- **mounts**: `[]`
- **triggers**: `[]`
- **spend**:
  - `cap`: Configured in `Application.get_env(:agent_os, :elicitor_spend_cap, 100_000)` (default: 100,000 micro-dollars / $0.10)
  - `window`: `:daily`
  - `on_breach`: `:kill`

## 2. Environment Variables

The Python elicitor process expects the following environment variables to be set by the substrate:

- `INFERENCE_SOCKET`: The absolute path to the Unix domain socket (e.g. `/Users/will/projects/agent_os/data/inference.sock`).
- `RUN_TOKEN`: The unique, session-specific opaque string generated by the `ElicitationSession`.

## 3. UDS Request/Response Schema

### Request (POST /v1/inference)
```json
{
  "run_token": "session_token_value",
  "model": "google/gemini-2.5-flash",
  "messages": [
    {
      "role": "system",
      "content": "..."
    },
    {
      "role": "user",
      "content": "..."
    }
  ]
}
```

### Response (200 OK)
```json
{
  "completion": "{\n  \"spec_draft\": {\n    \"purpose\": \"...\",\n    \"capabilities\": [...]\n  },\n  \"next_question\": \"...\"\n}"
}
```

## 4. Spend Ledger State

The spend ledger tracks spend under the `"elicitor"` key:
```elixir
%{
  "elicitor" => %{
    spent: 450,
    window_start: ~U[2026-07-01 12:00:00Z]
  }
}
```
