<objective>
Build the BEAM↔Python boundary: an Erlang `Port` that opens a stdin-guard wrapper, feeds the agent input on stdin, collects its JSON stdout, and surfaces its exit status cleanly — with a timeout so a hung agent does not block forever and no orphaned OS process survives a BEAM crash. Upgrade the stub Python agent into the human-written discovery agent that reasons over input and proposes enumerated actions.

Purpose: Implements REQ-hand-input (one port → human-written Python agent), REQ-reason-over-input (LLM reasons over input, unsanitized at v0), and REQ-propose-enumerated-actions. This is the load-bearing cross-boundary-failure seam (clean exit, no orphans) the whole architecture depends on. Per RESEARCH, plain Erlang `Port` + a tiny `port_wrapper.sh` — NOT erlexec/MuonTrap (Phase 2).

Output: `priv/port_wrapper.sh`, `lib/agent_os/port_runner.ex` (open/feed/collect/await/timeout), the discovery `agents/discovery/main.py` (run-once, emit enumerated JSON actions), and tests on both sides.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| BEAM → Python (the port) | The Python agent is UNTRUSTED. Input crosses on stdin; only JSON actions cross back on stdout; exit status is the clean failure signal. |
| OS process lifecycle | A spawned Python process is loosely coupled to the port; closing stdin must tear it down (no orphans). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-1-02 | Elevation of Privilege / Repudiation | orphaned Python process | mitigate | `priv/port_wrapper.sh` stdin-guard kills the child with `kill -KILL` when stdin closes (port close or BEAM death). PortRunner's timeout branch calls `Port.close/1`, which closes stdin and triggers the kill. Phase-gate manual check: `kill -9` the beam mid-run, assert `pgrep -f main.py` is empty. |
| T-1-03 | Denial of Service | hung Python agent | mitigate | PortRunner `receive ... after timeout_ms -> Port.close; {:error, :timeout}` (default 30s) prevents an infinite block; timeout is treated as abnormal exit (feeds restart-once in Plan 05). Verified by the `sleep 5` + `timeout_ms: 200` test. |
| T-1-09 | Tampering (prompt injection of input) | unsanitized stdin to agent | accept | v0 input is hand-fed and UNSANITIZED BY DESIGN (REQ-reason-over-input R1). Prompt-injection defense (input sanitization + isolation) is explicitly Phase 2. Documented, accepted, known gap. |
| T-1-10 | Information Disclosure | secrets via argv/env to the child | mitigate | No real credential exists at v0 (credential proxy is Phase 3); the agent command/args carry no secret. Convention established: never pass secrets via argv (RESEARCH Security V12). |
</threat_model>
