<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| run pipeline → substrate state/log | The RunWorker orchestrates untrusted agent output through the deterministic check + effector; all effects + traces are substrate-written, never agent-written. |
| supervisor → failure handling | A misbehaving/crashing child must surface cleanly (restart-once) and, on giving up, alert observably rather than failing silent. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-1-01 | Elevation of Privilege | ungranted agent action reaching effect | mitigate | RunWorker routes ALL agent actions through `OutputCheck.validate` before `Effector.act_all`; ungranted actions are dropped (Plan 04) and never reach an effect. Verified by the ungranted-dropped pipeline test. |
| T-1-12 | Denial of Service / Repudiation | child crash-loop or silent give-up | mitigate | Restart-once (`:transient` + max_restarts 1 / explicit retry) prevents an infinite restart loop; on the second failure `Alerter.alert/1` fires AND records an observable ALERT run-log line (legible give-up, not silent). All three exit paths tested (Pitfall 6). |
| T-1-13 | Repudiation | run with no trace | mitigate | Every run (success OR error) appends a timestamped legible run-log line (DEC-legibility-non-negotiable); the standing inventory reflects last-run state. No run is invisible. |
| T-1-02 | Elevation of Privilege | orphaned child on crash | mitigate | Inherited from Plan 03 (stdin-guard wrapper + PortRunner timeout). Phase-gate manual check: `kill -9` beam mid-run, `pgrep -f main.py` empty. |
</threat_model>
