<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| agent run → StateStore | The (untrusted) agent run must never mutate state directly; it receives a snapshot copy and all mutation funnels through the single-writer GenServer mailbox. |
| disk → StateStore | Term-file is read at init via `binary_to_term`; treated as trusted (substrate-owned, gitignored), not attacker-controlled at v0. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-1-04 | Tampering | StateStore (mounted state) | mitigate | Single-writer GenServer is the SOLE mutator; `snapshot/0` returns a copy (BEAM message-pass semantics), so a caller mutating the returned map cannot affect the store. `apply_action/1` is the only write path; malformed actions are rejected without mutating. Test asserts snapshot-mutation isolation. |
| T-1-04b | Tampering | term-file persistence | mitigate | Atomic write (tmp + `File.rename!`) prevents a torn/partial roster file on crash mid-write. File is gitignored and substrate-owned. |
| T-1-05 | Information Disclosure | Manifest contents | accept | At v0 the manifest IS human-readable and intentionally tracked in git (legibility). Manifest-not-agent-readable (DEC-manifest-not-readable-by-agent) is a Phase 3/4 enforcement concern; at v0 the agent does not receive the manifest as input. Documented accepted gap. |
</threat_model>
